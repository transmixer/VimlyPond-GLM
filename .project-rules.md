# 项目规则

## 环境使用规则
- **优先使用 WSL**：当系统已配置 WSL 时，优先使用 WSL 环境而非 PowerShell
- **WSL 配置**：默认分发为 Ubuntu，版本为 2

## 开发流程规则
- 严格按照 PR 流程进行代码提交和合并
- 实现新功能时使用 TDD (Test-Driven Development) 方法
- 修复问题后必须进行回归测试
- 确保所有单元测试通过

## 代码规范
- 使用 TypeScript 进行开发
- 遵循项目现有的代码风格和命名规范
- 保持代码清晰、可维护

## 构建和部署
- 确保项目能够正常构建
- 解决所有依赖问题
- 确保开发服务器能够正常运行
