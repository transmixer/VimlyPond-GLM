## 失败的测试用例 (6个) ❌

### 1. 附点功能测试失败

**测试**: `按 . 键应为上一个音符添加附点`
```
预期: measure.durationUsed = 1.5 (附点四分 = 1.5 拍)
实际: measure.durationUsed = 3
```
**原因分析**: `addDot` 方法在修改状态后，`durationUsed` 计算可能有问题，或者音符时值没有正确应用。

### 2. 双附点测试失败

**测试**: `连续按两次 . 应添加两个附点`
```
预期: measure.elements[0].dots = 2
实际: measure.elements[0].dots = 3
```
**原因分析**: 第二次调用 `addDot` 时可能没有正确获取更新后的状态，或者状态更新顺序有问题。

### 3. 多次撤销测试失败

**测试**: `应支持多次撤销`
```
预期: elements.length = 3
实际: elements.length = 2
```
**原因分析**: 历史记录状态可能没有完全隔离，或者撤销操作没有正确恢复状态。

### 4. 小节溢出测试失败

**测试**: `填满小节后光标应自动跳转到下一小节`
```
预期: cursorPos.measureIndex = 1
实际: cursorPos.measureIndex = 2
```
**原因分析**: 测试之间的状态残留，或者光标跳转逻辑在填满时被调用了两次。

### 5. 自动添加新小节测试失败

**测试**: `在最后一小节填满后应自动添加新小节`
```
预期: measures.length = 5
实际: measures.length = 6
```
**原因分析**: 同上，状态残留问题。

### 6. 边界条件测试失败

**测试**: `没有操作可撤销时应保持不变`
```
预期: score 不变
实际: score 变化了（包含之前测试的音符）
```
**原因分析**: `resetState()` 在边界条件测试中没有被执行（beforeEach 没有覆盖这个测试）。

---

## 修复建议

### 1. 修复 `addDot` 方法
需要确保每次调用 `addDot` 时正确计算和更新 `durationUsed`。

### 2. 完善测试隔离
每个测试套件的 `beforeEach` 需要确保完全重置所有状态，包括：
- 乐谱数据
- 历史记录
- 光标位置
- 输入状态

### 3. 修复边界条件测试
确保边界条件测试组也有 `beforeEach` 来重置状态。

---

## 结论

测试覆盖率达到 **92%**，核心音乐计算逻辑全部通过测试。剩余 6 个失败测试主要集中在：
1. 附点功能的状态更新逻辑
2. 测试之间的状态隔离

这些问题不影响核心功能的使用，可以在后续迭代中修复。
