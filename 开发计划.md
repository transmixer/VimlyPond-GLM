# 乐谱编辑器开发计划

## 🚨 PR流程规矩（AI必须遵守）

每次开发任何功能，必须严格按以下流程执行：

### 第一步：提议功能（新增环节）
- 在“提议中功能”区域添加一个新任务
- 必须包含：功能描述、为什么要做、技术方案、预估影响范围、测试要点
- 等待我确认后才能开始开发

### 第二步：建分支（确认后）
- 从 `main` 分支创建新分支
- 命名规则：`feature/功能名`

### 第三步：写代码
- 在分支上开发，遵循TDD：先写测试，再写代码
- 代码写完本地跑一遍测试，确保全绿

### 第四步：发起PR
- 在GitHub上发起Pull Request
- 标题格式：`[功能] 简短描述`
- PR描述必须包含：做了什么、怎么测试、影响范围

### 第五步：等待验收
- 等待GitHub Actions自动测试通过
- 等待我手动测试验收
- 我验收通过后，我会在GitHub上点击合并

### 禁止事项
- ❌ 禁止直接向 `main` 分支推送代码
- ❌ 禁止未经我确认就开始开发新功能
- ❌ 禁止测试未通过时请求合并
